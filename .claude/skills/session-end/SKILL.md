# セッション終了ワークフロー

以下を順に実施すること：

## 1. 実施内容の確認

このセッションで行ったことを振り返る：
- 変更・追加・削除したファイル
- 確定した仕様
- 新しく追加した機能・概念

## 2. specs/ の更新

仕様に変更・確定事項があれば該当ファイルを Edit で更新する：

- 新機能を実装した場合：対応する specs/ ファイルにパラメータ・動作仕様を反映
- 仕様が確定した場合：`specs/undecided.md` から確定済みに移動
- 新しい決定事項があれば記録
- 必要なら新しい仕様書を作成

**Edit ツールで差分編集すること。フル書き換え禁止。**

## 3. CLAUDE.md の実装状況テーブル更新

`CLAUDE.md` の「実装済みファイル」セクションを Edit で更新する：

- 新規ファイル：テーブルに追加
- 既存ファイル更新：概要欄の説明を最新の機能に更新
- 削除・リネーム：テーブルから削除または修正

**Edit ツールで差分編集すること。フル書き換え禁止。**

## 4. 整合性・重複チェック

以下の観点でチェックし、問題があれば修正する：

- CLAUDE.md の記述と specs/ の記述で矛盾がないか
- specs/ 間で矛盾する記述がないか
- 実際のコードの定数値と仕様書の記載が一致しているか
- 同じ内容が複数の仕様書に重複して書かれていないか

## 5. セッション完了メッセージを出力

以下の形式で締めメッセージを出力する：

```
✅ セッション完了なのだ！
📝 [更新したファイル名リスト] を更新したのだ。
👉 次のセッションでは「[次にやること]」から始めるのだ！
新しいセッションを開始してほしいのだ🥜
```

次にやることが不明な場合は、このセッションの作業から自然につながる次のステップを提案すること。
