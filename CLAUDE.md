# Flowmaid - フローチャート to Mermaid エディタ

## プロジェクト概要

自由配置可能なフローチャートエディタ。
図形をドラッグ&ドロップで自由に配置しながら、リアルタイムでMermaid記法が生成される。

## 技術スタック

- **フレームワーク**: Next.js 16 + TypeScript (strict)
- **フローチャートライブラリ**: React Flow (@xyflow/react v12)
- **UIコンポーネント**: shadcn/ui（Tailwind CSS v4 + Radix UI）
- **状態管理**: zustand + zundo（undo/redo、50履歴）
- **国際化**: 自前実装（zustand + 辞書、ja/en）
- **テーマ**: next-themes（ダーク/ライト）
- **シリアライズ**: yaml（.flowmaid フォーマット）
- **テスト**: vitest + jsdom（70テスト）

## 出力フォーマット (.flowmaid)

エディタの状態は以下の独自フォーマットで出力される。2つのセクションからなるテキストファイル。

```
--- mermaid ---
graph TD
    A[開始] --> B{条件分岐}
    B -->|Yes| C(処理A)
    B -->|No| D((処理B))
    C --> E([終了])
    D --> E
    subgraph comp1_認証処理[認証処理]
        comp1_n1[Start]
        comp1_n2[ログイン]
        comp1_n3[End]
        comp1_n1 --> comp1_n2
        comp1_n2 --> comp1_n3
    end

--- layout ---
direction: TD
nodes:
  A:
    position: { x: 100, y: 50 }
    size: { width: 150, height: 50 }
    shape: rectangle
    label: 開始
  B:
    position: { x: 100, y: 150 }
    size: { width: 100, height: 100 }
    shape: diamond
    label: 条件分岐
    fillColor: "#fffde7"
    borderColor: "#ef4444"
    borderWidth: 4
    borderStyle: dashed
    zIndex: 2
  F:
    position: { x: 300, y: 50 }
    size: { width: 200, height: 160 }
    shape: rectangle
    label: 認証処理
    componentDefinitionId: comp1
    componentInstanceName: 認証処理
    componentSyncVersion: 1
edges:
  A-B-bottom-source-top-target:
    source: A
    target: B
    sourceHandle: bottom-source
    targetHandle: top-target
    label: Yes
    edgeType: step
    markerEnd: arrowclosed
    markerStart: none
    strokeWidth: 4
    strokeColor: "#3b82f6"
    strokeStyle: dashed
componentDefinitions:
  - id: comp1
    name: 認証処理
    version: 1
    entryNodeId: n1
    exitNodeId: n3
    nodes:
      - id: n1
        label: Start
        shape: stadium
        position: { x: 50, y: 0 }
      - id: n2
        label: ログイン
        shape: rectangle
        position: { x: 50, y: 120 }
      - id: n3
        label: End
        shape: stadium
        position: { x: 50, y: 240 }
    edges:
      - id: e1
        source: n1
        target: n2
      - id: e2
        source: n2
        target: n3
```

### `--- mermaid ---` セクション
- 標準的なMermaid記法（`graph TD` / `graph LR`）
- 他のMermaidツールにそのままコピペで使える
- 対応ノード形状: `[]` 処理, `{}` 判断, `()` 代替処理, `(())` 結合子, `[//]` データ, `[()]` データベース, `{{}}` 準備, `([])` 端子, `[\\]` 手操作, `[/\]` 書類, `[[]]` 定義済み処理, `[\/]` 手入力
- `text`（フリーテキスト）、`internalStorage`（内部記憶）、`display`（表示）はMermaid固有構文がないため、出力時は矩形/角丸にフォールバック

### `--- layout ---` セクション（YAML）
Mermaid記法では表現できないビジュアル情報:

#### nodes の各プロパティ

| プロパティ | 必須 | 説明 | 例 |
|---|---|---|---|
| `position` | ✅ | キャンバス上の座標 | `{ x: 100, y: 50 }` |
| `size` | | ノードの幅・高さ | `{ width: 150, height: 50 }` |
| `shape` | ✅ | 形状（15種類） | `rectangle`, `diamond`, `roundedRect`, `circle`, `parallelogram`, `cylinder`, `hexagon`, `stadium`, `trapezoid`, `document`, `predefinedProcess`, `manualInput`, `internalStorage`, `display`, `text` |
| `label` | ✅ | 表示テキスト | `開始` |
| `fillColor` | | 塗りつぶし色 | `"#ef4444"` |
| `borderColor` | | 枠線色 | `"#3b82f6"` |
| `borderWidth` | | 枠線太さ（デフォルト: 2） | `1`, `2`, `4` |
| `borderStyle` | | 枠線スタイル（デフォルト: solid） | `solid`, `dashed`, `dotted` |
| `zIndex` | | 表示順序（大きいほど前面） | `2` |
| `fontSize` | | 文字サイズ（デフォルト: 14） | `10`, `14`, `18`, `24` |
| `textColor` | | 文字色 | `"#ef4444"` |
| `textAlign` | | 文字位置（デフォルト: center） | `left`, `center`, `right` |
| `bold` | | 太字 | `true` |
| `italic` | | 斜体 | `true` |
| `underline` | | 下線 | `true` |
| `fillOpacity` | | 塗りつぶし透明度（0=透明〜10=不透明、デフォルト: 10） | `5` |
| `fillLightness` | | 塗りつぶし明暗（0=明るい〜10=暗い、デフォルト: 5） | `3` |
| `borderOpacity` | | 枠線透明度（0=透明〜10=不透明、デフォルト: 10） | `5` |
| `borderLightness` | | 枠線明暗（0=明るい〜10=暗い、デフォルト: 5） | `3` |
| `textOpacity` | | 文字透明度（0=透明〜10=不透明、デフォルト: 10） | `5` |
| `textLightness` | | 文字明暗（0=明るい〜10=暗い、デフォルト: 5） | `3` |
| `componentDefinitionId` | | コンポーネント定義ID（インスタンスノードのみ） | `"comp1"` |
| `componentInstanceName` | | インスタンス名 | `"認証処理"` |
| `componentSyncVersion` | | 同期済みバージョン | `1` |
| `collapsed` | | 折りたたみ状態 | `true` |
| `expandedSize` | | 折りたたみ前の展開サイズ（collapsed時に保持） | `{ width: 200, height: 160 }` |

#### edges の各プロパティ

| プロパティ | 必須 | 説明 | 例 |
|---|---|---|---|
| `source` | ✅ | 接続元ノードID | `A` |
| `target` | ✅ | 接続先ノードID | `B` |
| `label` | | エッジのラベル | `Yes` |
| `sourceHandle` | | 接続元ハンドル位置 | `bottom-source` |
| `targetHandle` | | 接続先ハンドル位置 | `top-target` |
| `edgeType` | | 線の種類（デフォルト: bezier） | `bezier`, `straight`, `step` |
| `markerEnd` | | 終点矢印（デフォルト: arrowclosed） | `arrowclosed`, `arrow`, `none` |
| `markerStart` | | 始点矢印 | `arrowclosed`, `arrow`, `none` |
| `strokeWidth` | | 線の太さ（デフォルト: 2） | `1`, `2`, `4` |
| `strokeColor` | | 線の色 | `"#ef4444"` |
| `strokeStyle` | | 線のスタイル（デフォルト: solid） | `solid`, `dashed`, `dotted` |
| `strokeOpacity` | | 線の透明度（0=透明〜10=不透明、デフォルト: 10） | `5` |
| `strokeLightness` | | 線の明暗（0=明るい〜10=暗い、デフォルト: 5） | `3` |

※ デフォルト値のプロパティはファイルサイズ節約のため出力を省略する

#### componentDefinitions セクション

コンポーネント定義の配列。インスタンスノードが参照するテンプレート情報。

| プロパティ | 必須 | 説明 | 例 |
|---|---|---|---|
| `id` | ✅ | 定義の一意ID | `"comp1"` |
| `name` | ✅ | 定義名 | `"認証処理"` |
| `version` | ✅ | バージョン番号（編集ごとにインクリメント） | `1` |
| `entryNodeId` | | 入口ノードID | `"n1"` |
| `exitNodeId` | | 出口ノードID | `"n3"` |
| `nodes` | ✅ | 内部ノード配列（id, label, shape, position） | |
| `edges` | ✅ | 内部エッジ配列（id, source, target, label?） | |

### Phase 2

- [x] エッジの経路点（waypoint）の手動編集
- [x] スナップ / ガイドライン
- [x] Mermaid記法からの逆変換（テキスト → フローチャート）
  - インポートダイアログでエッジタイプ（ベジェ/直線/折れ線）を選択可能
  - 選択値は `localStorage` (`flowmaid-mermaid-edge-type`) に永続化
  - キャンバスに既存コンテンツがある場合、変換時に上書き確認ダイアログを表示
- [x] コンポーネント（再利用可能テンプレート方式）
  - コンポーネント定義（テンプレート）を登録し、キャンバスにインスタンスとして複数配置
  - 定義編集は専用モード（メインキャンバスを一時退避して切替、テーマ反転フレーム表示）
  - 定義にはEntry/Exitノード（ロック付き、編集・削除不可）を含む
  - インスタンスは展開/折りたたみ、インスタンス名変更が可能（子ノードはプレビュー専用: 選択・ドラッグ不可）
  - 定義編集モード終了時にインスタンスを自動同期（確認ダイアログなし）
  - 変更検知は`.flowmaid`シリアライズ形式で比較（変更なしなら保存・バージョンアップなし、新規未編集なら削除）
  - Mermaid出力では `subgraph` として展開出力（Start/End含む完全な内部構造を定義から直接取得）
  - `.flowmaid`エクスポート/インポートで定義+インスタンス情報を保持
  - TD/LR方向切替に対応（編集モード内でもノード座標・エッジハンドルを自動調整）
  - ブラックボックス表示: Start/Endノードはインスタンス上で非表示、外部エッジは親ノードに接続しブリッジエッジで内部ノードに中継
  - 方向対応コネクタ: 入口・出口の両ポジションにsource/target各ハンドルを配置（双方向接続対応）
  - ブリッジは自然方向のみ生成: 入口=incoming、出口=outgoing（逆方向はブリッジなし）
  - 折りたたみ時は150x50の通常ノードサイズに縮小（[+]ボタン付き）、展開時に元のサイズに復元
  - ブリッジエッジは直線（straight）固定、シリアライズ時はスキップ（実行時に再生成）
- [x] エッジ・コネクタのUX改善
  - エッジ選択時のグロー: primary色（テーマ連動）、太めのグロー表示
  - 空ラベルのエッジはラベルボックス非表示（選択時は「+」プレースホルダー表示）
  - エッジの線ダブルクリックでラベル編集開始（step以外。stepはwaypoint追加に使用）
  - リングハンドルによるエッジ付け替え（接続済みハンドルにリング⊙表示、ドラッグで別ノードへ再接続）
  - リング表示条件: エッジ選択時の両端ハンドル OR ノードホバー時の接続済みハンドル
  - 接続ハンドル（ドット）の表示条件: ノードホバー時のみ（ノード選択時は非表示 → パフォーマンス最適化）
  - コネクションライン方向: 付け替え中は固定端（変更しない側）からカーソルへ向かって描画
  - ハンドルID正規化: React Flowのsource/targetスワップに対応し、再接続時のハンドルIDサフィックスを正しい型に変換
  - ノードハンドルの不可視ヒットエリア拡大（30px透明ラッパー、ズーム補正対応）
  - ハンドルドット・ヒットエリアのズーム補正（`calc(size / var(--rf-zoom, 1))`）
  - React Flowデフォルトの EdgeUpdateAnchors を非表示化（リングハンドルに統一）
- [x] コンポーネントインスタンスのリサイズ時に子ノードを比例スケーリング
  - 親ノードリサイズ時にコンテンツ領域（ヘッダー・パディング除外）の比率で子ノード位置・サイズを自動調整（アスペクト比維持: `Math.min(scaleX, scaleY)`）
  - ユーザーによる子ノードの直接リサイズは不可
- [x] 折れ線エッジのノード回避ルーティング
  - 折れ線（step）の接続セグメントがソース/ターゲット以外のノードと重なる場合、midY/midXを自動シフトして回避
  - マージン20px、最大5回の反復調整でノード群を回避
  - 基本パス（ウェイポイント2個以下）のみ自動回避、手動カスタマイズ（3個以上）はそのまま維持
- [x] ノード形状の追加（5種類）

  - Tab/Shift+Tab でテーブル行間のキーボードナビゲーション
  - コンポーネントインスタンス対応（instanceName表示、ロック/ブリッジノードは除外）
  - ノード/エッジアイコン表示（形状・色・マーカーを反映したSVGミニアイコン）
- [x] ミニマップ表示/非表示トグル

#### サイドパネル共通の動作
ノードパレット（左）とMermaidプレビュー（右）は同じ開閉メカニズムを持つ。

- **リボン（タブ）**: パネルのキャンバス側端に常時表示される細い帯。矢印アイコンで開閉状態を示す
- **クリックで開閉**: リボンをクリックするとスライドアニメーションで表示/非表示
- **非表示時**: パネルが畳まれ、キャンバス領域がその分広くなる
- **両方非表示**: キャンバスが最大化され、左右端にリボンのみ表示
- **ドラッグリサイズ**: Mermaidプレビュー表示時、キャンバスとの境界をドラッグで幅調整可能
- **状態保持**: 各パネルの開閉状態・幅はlocalStorageに保存し、次回起動時に復元

### コンポーネント編集モード

メインキャンバスの内容を一時退避し、コンポーネント定義の内部ノード/エッジを専用キャンバスで編集する。

```
┌─────────────────────────────────────────────────────────────┐
│ ■ テーマ反転フレーム（ライト→暗、ダーク→明）                   │
│                                                             │
│  コンポーネント 編集中 ( 認証処理 )  ● 接続済み  [戻る]      │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │                                                         │ │
│ │  🔒 Start  ──→  ログイン  ──→  🔒 End                  │ │
│ │                                                         │ │
│ │  通常キャンバスと同じ操作（ノード追加、エッジ接続等）     │ │
│ │  ※ Start/End ノードはロック（編集・削除不可）            │ │
│ │                                                         │ │
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

- **ヘッダー**: 編集中コンポーネント名、Start→End接続状態（緑:接続済み / 赤:未接続）、戻るボタン
- **フレーム**: テーマ反転色の枠（`var(--foreground)`背景）、CSSトランジションでアニメーション切替
- **Entry/Exitノード**: `isLocked`フラグで保護（ラベル変更不可、削除不可、🔒アイコン表示）
- **戻るボタン**: 接続済み→保存して戻る（全インスタンス自動同期）/ 未接続→破棄確認ダイアログ
- **変更検知**: `.flowmaid`シリアライズで開始時スナップショットと比較。変更なし→保存スキップ。新規v1で未変更→定義削除
- **自動同期**: 保存時に全インスタンスの子ノード・ブリッジエッジを自動再生成（確認ダイアログなし）
- **自動保存**: 編集モード中はメインフローのlocalStorage自動保存を停止

### テキスト一括編集（BulkEdit）モード

ノード/エッジのラベルをテーブルUIで一括編集するモード。左に読み取り専用キャンバス、右に編集テーブル。

```
+-------------------------------------------------------------+
|  ツールバー  [← 戻る]                                        |
+-------------------------------+|+----------------------------+
|                               ||  [検索...] [All shapes ▼]  |
|  読み取り専用キャンバス         ||  [Category / Flow]         |
|  （独立ReactFlowProvider）     ||                            |
|                               || ■ Nodes                    |
|  ・左ドラッグ → 選択ボックス    ||   [□] ノードA  [____]      |
|  ・右/中ドラッグ → パン         ||   [◇] ノードB  [____]      |
|  ・Ctrl+クリック → 個別トグル   || ■ Edges                    |

- **キャンバス**: 読み取り専用（ドラッグ・接続不可）、独自の選択管理、ミニマップ非搭載

- **エッジ選択**: 両端点ルール（ドラッグ選択時、ソースとターゲットの両ノードが選択領域内にある場合のみエッジを自動選択）
- **フォーカス**: テーブル行クリック → キャンバスでハイライト+ズーム、キャンバスクリック → ハイライトのみ

### 差分比較モード

2つの .flowmaid ファイルをインポートし、ファイルAをベースに変化点を1画面に統合表示する（Git diff的アプローチ）。
左右パネルの境界はドラッグで自由にリサイズ可能（リサイザブルスプリッター）。

```
+-------------------------------------------------------------+
|  ツールバー  [ファイルA選択] [ファイルB選択] [比較実行]        |
+-------------------------------+||+---------------------------+
|                               |||                           |
|  キャンバス（統合差分表示）     |||  テキスト差分パネル        |

#### 左パネル: ビジュアル差分
- ファイルAのレイアウトをベースにフローチャートを描画
- 差分箇所をハイライト色で視覚的に表示
- ハイライトされたノード/エッジをクリックすると、右パネルの該当箇所にスクロール

#### 右パネル: テキスト差分
- 変更内容を一覧でテキスト表示（追加/削除/変更/移動をアイコン+色で区別）
- 各変更項目の詳細（変更前 → 変更後）を表示
- 項目クリックで左パネルの該当ノード/エッジにフォーカス移動
- Mermaid記法のテキストdiffへの表示切替も可能

#### 差分検出カテゴリ

| 変化の種類 | キャンバス上の表示 | 検出対象 |
|---|---|---|
| **追加** | 緑のハイライト/枠線 | 新しく追加されたノード・エッジ |
| **削除** | 赤のハイライト/枠線（半透明） | 削除されたノード・エッジ |
| **テキスト変更** | 黄のハイライト/枠線 | ラベルやテキストの変更 |
| **位置移動** | 青のハイライト + 移動矢印 | 座標が変わったノード（layout情報で検出） |
| **接続変更** | 黄のハイライト | エッジの接続先が変わった |
| **未変更** | 通常表示 | 変化なし |

## 仕様詳細

### Mermaid記法の方向
- **デフォルト**: `graph TD`（上から下）
- **切り替え可能**: UI上でTD（上→下）/ LR（左→右）を切り替え可能
- 方向の変更はMermaid出力にのみ影響し、キャンバス上の自由配置には影響しない

### ノードID
- **自動連番**: A, B, C, ... Z, AA, AB, ... の形式で自動生成
- ユーザーがID を変更可能（Mermaid出力の可読性のため）
- ID の一意性はバリデーションで保証
- 差分比較ではこのIDを基準にノード間のマッチングを行う

### ノード追加方法
- **ドラッグ&ドロップ**: パレットからキャンバスにドラッグして任意の位置に配置（シングルクリック配置は廃止済み）

### テーマ
- **ライト/ダークモード**: トグルで切り替え可能
- キャンバス、パネル、ツールバー全体に適用
- ユーザーの選択はlocalStorageに保存
- システム設定（prefers-color-scheme）をデフォルトとして使用

### コンポーネント

再利用可能なテンプレート方式のサブフロー機能。定義を登録し、キャンバスにインスタンスとして何度も配置できる。

#### データモデル
- **ComponentDefinition**: テンプレート定義（内部ノード群 + 内部エッジ + 入口/出口ノード指定 + 名前 + バージョン番号）
- **ComponentInstance**: キャンバス上のインスタンスノード（`type: "componentInstance"`、定義ID参照）

#### 作成・編集フロー
- サイドパネル（ノードパレット下部）の「+新規」ボタンから作成
- 新規作成時は簡易テンプレート（Start→Process→End）が自動生成され、即座にコンポーネント編集モードに入る
- 定義はメインキャンバスと切り替わる専用編集モードで編集
- コンテキストメニューからの「コンポーネントとして登録」機能は廃止済み

#### インスタンス
- 管理パネルの「キャンバスに配置」ボタンまたはドラッグ&ドロップでビューポートに配置
- インスタンスごとに名前を付けられる（展開時: ヘッダーダブルクリック、折りたたみ時: ラベルダブルクリックで編集）
- 子ノードはプレビュー専用（`selectable: false`, `draggable: false`）、ラベルは定義のものをそのまま表示
- 展開/折りたたみ トグル（ヘッダー右端に [−]/[+] ボタン）
- スタイル変更対応: 塗りつぶし色・枠線色・枠線スタイル・フォント設定が通常ノードと同様に変更可能

#### ブラックボックス表示
- Entry(Start)/Exit(End)ノードはインスタンスのキャンバス上で非表示（子ノードとして生成されない）
- Entry/Exitに接続されていた内部エッジも除外（定義には残る）
- 外部エッジは常に親ノード（コンポーネントインスタンス）に接続
- **ブリッジエッジ**: 外部エッジが親ノードに接続されると、自動的に内部の子ノードへ直線（straight）で中継
  - 自然方向のみ生成（2パターン）:
    - 入口incoming: 外部→親ノード入口target → bridge-entry-source → entryTarget子ノード
    - 出口outgoing: exitSource子ノード → exit-source → 親ノードbridge-exit-target → 外部
  - 逆方向の接続（入口からの出発、出口への到着）にはブリッジを生成しない
  - ハンドル位置が不明（null）のエッジにはブリッジを生成しない
  - ブリッジエッジは `data.isBridgeEdge = true` で識別、`selectable: false`、シリアライズ時はスキップ（デシリアライズ/ロード時に再生成）
- **方向対応コネクタ**: 入口・出口の両ポジションにsource/targetの4ハンドルを配置
  - TD: 上側(source+target) + 下側(source+target)
  - LR: 左側(source+target) + 右側(source+target)
  - 隠しブリッジハンドル: `bridge-entry-source`（入口位置）+ `bridge-exit-target`（出口位置）、`isConnectable={false}`

#### 折りたたみ
- 折りたたみ時: `expandedSize` に現在サイズを保存、親ノードを150x50に縮小、子ノードとブリッジエッジをhidden
- 展開時: `expandedSize` からサイズ復元（なければ `calculateComponentSize` で計算）、子ノードとブリッジエッジを表示
- 折りたたみ表示: 枠線色・塗りつぶし色はスタイル設定に従う（デフォルト: `--muted-foreground`枠線 + `--muted`背景）+ [+]ボタン + コンポーネント名（ダブルクリックで編集可能）

#### 自動同期
- 定義編集モード終了時（`exitComponentEditMode`）に変更を検知し、全インスタンスを自動同期
- `componentSyncVersion` を定義の `version` に自動更新
- 確認ダイアログなし（プレビュー専用のため即時反映）

#### 入口/出口
- 定義内でEntry/Exitノードを明示指定（`isLocked`フラグで保護）
- インスタンスではEntry/Exitは非表示（ブラックボックス）、外部エッジはブリッジエッジ経由で内部ノードに接続

#### Mermaid出力
- インスタンスは `subgraph ID[ラベル] ... end` として展開出力
- subgraph内のノード/エッジは定義から直接取得（Start/End含む完全な内部構造を出力）
- 内部ノードIDは `{インスタンスID}_{内部ID}` で一意化
- ブリッジエッジ（`isBridgeEdge`）はMermaid出力から除外

#### 管理パネル（左パネル「コンポーネント」タブ）
- 定義一覧（名前ダブルクリックでリネーム、バージョン番号表示、ドラッグ&ドロップでキャンバスに配置）
- 「編集」→ コンポーネント編集モード
- 「×」→ 使用中インスタンス確認後に定義削除
- 一覧末尾に点線枠のダミーカード（クリックで新規作成）
- コンポーネント編集モード中はパネル非表示

### エッジ付け替え（リングハンドル方式）

エッジの接続先を変更する統一的なUX。React Flowデフォルトの EdgeUpdateAnchors（エッジ線ドラッグ）は非表示にし、リングハンドルに一本化。

#### 接続ハンドル（ドット）の表示条件
- **ノードホバー時のみ**: マウスがノード上にあるとき、4方向の接続ハンドルを表示
- **ノード選択時**: 接続ハンドルは表示しない（パフォーマンス最適化のため）

#### リングハンドルの表示条件
- **エッジ選択時**: 選択されたエッジの両端ハンドルにリング（⊙）+ primary色グロー表示
- **ノードホバー時**: ホバー中のノードで接続済みのハンドルにリング表示
- リング表示中のハンドルは `zIndex: 10` で同位置の非接続ハンドルより前面に配置
- リング表示中のカーソルは常にgrab

#### ドラッグ操作
- **リングハンドルからのドラッグ（ハンドルにエッジ1本のみ）** → エッジ付け替え（reconnect）
  - エッジ選択の有無に関係なく、接続済みハンドルのドラッグで付け替え開始
  - ソースハンドルのリングをドラッグ → ソース側を変更、ターゲット側は固定
  - ターゲットハンドルのリングをドラッグ → ターゲット側を変更、ソース側は固定
- **ハンドルに複数エッジがある場合** → 新規エッジ作成（どのエッジを付け替えるか曖昧なため）
- **未接続ハンドルからのドラッグ** → 新規エッジ作成（同じハンドルから複数エッジ可）
- ドラッグ中のコネクションラインは固定端からカーソルに向かって描画（`ReconnectConnectionLine` カスタムコンポーネント）
- ドラッグ中は元のエッジを非表示（CSS `.reconnecting { visibility: hidden }`）

#### ハンドルID正規化
- React Flowは `ConnectionMode.Loose` + target ハンドルからのドラッグ時に connection オブジェクトの source/target を入れ替える
- **新規エッジ作成・再接続の両方**で、sourceHandleは常に`-source`サフィックス、targetHandleは常に`-target`サフィックスに正規化

#### その他のエッジUX
- `connectOnClick={false}`: クリックによるエッジ接続を無効化（ドラッグのみ）
- 折れ線（step）エッジのwaypoint角丸: 二次ベジェ曲線（`Q`コマンド）で `WAYPOINT_BORDER_RADIUS=8` の角丸を適用
- 折れ線エッジのノード回避: 接続セグメントがノードと重なる場合にmidY/midXを自動シフト（`NODE_AVOID_MARGIN=20`、最大5回反復）

#### 実装上の注意点
- zustandセレクタ内で他のセレクタ結果（クロージャ変数）を参照してはいけない — 独立サブスクリプション間のstale値問題を起こす
- 各ノード位置に source/target 2つのハンドルが重なるため、リング（接続済み）ハンドルの `zIndex` を上げて正しいハンドルがドラッグされるようにする

### 選択操作（useCtrlSelection）

FlowCanvasとBulkEditCanvasで共有される選択ロジック（`src/hooks/useCtrlSelection.ts`）。

#### 通常ドラッグ選択
- 左ドラッグで選択ボックスを描画、`SelectionMode.Partial`でノードを選択
- エッジは両端点ルール: ソースとターゲットの両ノードが選択されている場合のみ自動選択（`syncEdgeSelection`）
- ReactFlowの`onEdgesChange`のselect変更はドラッグ中は無視（`processEdgesChanges`で即return）

#### Ctrl+クリック
- ノード/エッジの個別トグル（選択⇔解除）
- ハイライト中の要素がある場合は起点として含める

#### Ctrl+ドラッグ（XOR選択）
- ドラッグ開始時のselection（`dragStartIds`）とドラッグ中のボックス内容の対称差（XOR）が結果
- `@xyflow/system`の`getNodesInside(nodeLookup, userSelectionRect, transform)`でボックス内ノードを直接計算
- `handleSelectionStart`でCtrl押下時に`rfStore.subscribe`を設置し、`userSelectionRect`変更のたびに`computeCtrlDragSelection`を呼び出し
- これによりReact Flow Paneの`areSetsEqual`最適化（前回ドラッグと同じボックス内容だと`getSelectionChanges`をスキップする問題）をバイパス
- `processNodesChanges`のCTRL+DRAG分岐でも同じ計算を実行（両方idempotentなので競合なし）
- `syncInternalSelection`でReact Flow内部の`nodeLookup`/`edgeLookup`の`.selected`を直接修復（`adoptUserNodes`のcheckEquality参照チェックで発生するstale値問題への対策）
- エッジも両端点ルールで自動同期

#### Ctrl+A（全選択）
- `getAllSelectableIds`で取得した全ID（ブリッジエッジ除外）を選択
- `useKeyboardShortcuts`にも同等の処理あり（FlowCanvas用）

#### パフォーマンス最適化
- `setsEqual`で変更がない場合は`setSelectedIds`を呼ばない（不要な再レンダリング防止）
- 全`useCallback`の依存配列は`[]`（状態は`optsRef`経由で参照）
- ドラッグ中のエッジselect変更は即returnで処理スキップ
- Ctrl+ドラッグ中のstore subscriptionは`handleSelectionEnd`で解除（非Ctrl+ドラッグ時はsubscriptionなし）

#### パフォーマンス特性（計測済み）
- 選択ロジック自体（`setSelectedIds`含む）は0.5ms未満で高速
- ドラッグ選択時の引っ掛かりはReact開発モードのオーバーヘッド（`logComponentRender`等で~30ms）が原因
- 本番ビルドでは引っ掛かりなし（開発モード固有の問題、対策不要）

# プロジェクト共通設定

## 基本ルール

- ファイル修正は **Edit ツールで差分編集**する（フル書き換え禁止）
- 修正前に対象ファイルを **Read して最新状態を確認**する
- マジックナンバーの直接記述は禁止（定数化する）
- 不明点があればユーザーに確認してから進む

## エージェント構成

| エージェント | モデル | 役割 |
|-------------|--------|------|
| project-explorer | haiku | プロジェクト探索・調査（読み取り専用） |
| bug-detective | sonnet | バグ原因調査（読み取り専用） |
| code-reviewer | sonnet | コードレビュー（読み取り専用） |
| feature-planner | sonnet | 機能設計・仕様書作成（specs/ への書き込みのみ） |

## スキル（ワークフロー）

| スキル | 用途 |
|--------|------|
| `/implement [機能名]` | 実装フロー：project-explorer → 仕様確認 → 実装 → code-reviewer |
| `/bug-fix [バグ概要]` | バグ修正フロー：bug-detective → 方針確認 → 修正 → code-reviewer |
| `/new-feature [機能名]` | 新機能追加フロー：仕様確認 → feature-planner → ユーザー承認 → 実装 |
| `/session-end` | セッション終了：specs/ 更新 → CLAUDE.md 更新 → 整合性チェック |
| `/permissions-review` | 権限レビュー：ツール使用ログ集計 → 現在の権限確認 → 追加ルール提案 |

## ワークフローの原則

1. **実装前に必ず調査する** — project-explorer で関連コードを把握してから着手
2. **実装後に必ずレビューする** — code-reviewer で品質チェック
3. **仕様は specs/ で管理する** — 仕様の確定・未決定事項は specs/ に記録
4. **セッション終了時にドキュメントを更新する** — `/session-end` で specs/ と CLAUDE.md を最新化
