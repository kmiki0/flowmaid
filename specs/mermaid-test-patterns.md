# Mermaid Import Test Patterns

Mermaid読み込み機能のテスト用記法集。ダイアログに貼り付けて動作確認に使う。

## 1. 全形状テスト (TD)

全9形状を網羅。ひし形からの分岐あり。

```
graph TD
    A[開始]
    B{条件分岐}
    C(処理A)
    D((終了))
    E([ターミナル])
    F[(データベース)]
    G{{準備}}
    H[/入力/]
    I[\出力\]
    A --> B
    B -->|Yes| C
    B -->|No| G
    C --> D
    G --> H
    H --> I
    I --> E
    E --> F
    F --> D
```

## 2. シンプルなフロー (LR)

LR方向 + サイクル（D -> B）あり。

```
graph LR
    A[Start] --> B{Is it working?}
    B -->|Yes| C[Great]
    B -->|No| D[Fix it]
    D --> B
```

## 3. チェーンエッジ

1行に `A --> B --> C` と連結するパターン。サイクルあり。

```
graph TD
    A[Step 1] --> B[Step 2] --> C[Step 3]
    C --> D{OK?}
    D -->|Yes| E((Done))
    D -->|No| A
```

## 4. エスケープ文字テスト

`escapeMermaid()` で変換された特殊文字が正しく復元されるか確認。

```
graph TD
    A[A #amp; B]
    B{X #lt; Y}
    C(#lbrack;配列#rbrack;)
    A --> B
    B --> C
```

## 5. 複雑なフロー

実用的なログイン認証フロー。10ノード、多段分岐、サイクルあり。

```
graph TD
    A[ユーザー入力]
    B{認証チェック}
    C[ダッシュボード]
    D[ログイン画面]
    E{権限確認}
    F([管理者ページ])
    G([一般ページ])
    H[(ログ保存)]
    I((完了))
    A --> B
    B -->|成功| C
    B -->|失敗| D
    D --> A
    C --> E
    E -->|管理者| F
    E -->|一般| G
    F --> H
    G --> H
    H --> I
```

## 確認ポイント

- [ ] 全9形状が正しい図形で表示される
- [ ] エッジラベル（Yes/No/成功/失敗 等）が保持される
- [ ] TD/LR 方向が正しく反映される
- [ ] チェーンエッジで中間ノードも正しく生成される
- [ ] サイクル（循環参照）があってもレイアウトが崩れない
- [ ] エスケープ文字が正しく復元される
- [ ] ノード間の余白が均等で、形状サイズに関わらず見やすい
